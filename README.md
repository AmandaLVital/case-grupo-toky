# Case Profissional – Estagiária de Dados no Grupo TOKY (Mobly + Tok&Stok)

Este repositório apresenta um resumo claro e profissional do meu trabalho como **Estagiária de Dados no Grupo TOKY**, formado pela união entre **Mobly** e **Tok&Stok**.

 *Nenhum dado, código, planilha, dashboard ou informação interna está sendo compartilhado, conforme política de confidencialidade.*  
 *Caso necessário, posso demonstrar todo o funcionamento e explicar os processos em detalhes durante a entrevista.*

---

##  Contexto do Trabalho

Atuo em um ambiente **pós-fusão**, onde Mobly e Tok&Stok ainda mantêm **bases de dados independentes**, estruturas diferentes e necessidades próprias.

Isso cria desafios como:

- duas arquiteturas de dados distintas  
- dados não unificados  
- processos e regras que variam entre as operações  
- relatórios duplicados, porém com lógicas diferentes  
- alto volume de demandas  
- necessidade de adaptação rápida  

Sou **a única estagiária do time Comercial/Produto**, atuando diretamente com análises, automações e pipelines desde o meu primeiro mês.

---

##  Principais Entregas e Impacto Real

###  1. Pipelines e ETL usando SQL + Python + Databricks
- Extração, transformação e modelagem de dados para ambas as empresas.
- Criação e manutenção de notebooks.
- Auditoria e validação da qualidade dos dados.
- Correção de falhas em jobs automáticos.
- Envio de dados para BigQuery e Sheets.

###  2. Automação da Consulta Interna de Produtos (Sheets + AppScript)
Problema original:

- 6 abas confusas, difíceis de usar.  
- Usuários sem clareza de onde clicar ou pesquisar.  
- Processo demorava ~10 minutos.

Minha solução:

- Reestruturei **toda a aplicação interna**.  
- Reduzi para **apenas 2 abas** (consulta + resultado).  
- Interface simples, intuitiva e limpa.  
- Pesquisa otimizada com AppScript.  
- Tempo de uso caiu para **1 minuto**.  
- Projeto muito elogiado internamente.

###  3. Central de Inteligência (Looker + Sheets + Notebooks)
Antes:

- Funcionários não conseguiam encontrar dashboards, planilhas ou notebooks.  
- Perdiam tempo procurando arquivos e, muitas vezes, deixavam de usá-los.

Solução:

- Criei uma Central com **todos os dashboards, sheets e notebooks da empresa**.  
- Implementação de busca por nome e descrição.  
- Aumento de produtividade e acesso à informação.

###  4. Dashboard de Gestão de Mostruário
Criado para lojas Mobly e Tok&Stok.

Permite:

- consultar estoque real e histórico  
- prever próximas entregas  
- identificar reposições futuras  
- evitar exposição de produtos sem estoque  
- substituir produtos com base na carteira de entrega

Resultado:

✔ funcionários passaram a tomar decisões melhores  
✔ exposição de produtos ficou mais eficiente  
✔ elogios de gerentes de loja

###  5. Análises e insights estratégicos
- Construção de dashboards no Looker.  
- Geração de insights comerciais.  
- Apoio a decisões sobre vendas, estoque e performance.  
- Construção de planilhas analíticas completas.
